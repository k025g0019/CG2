このリファレンスは「どのライブラリを使っているか」ではなく、 「そのコンポーネントを何に使うか」「Play で何が動くか」「今どの完成度か」を確認するための入口です。 Unity 風に GameObject へコンポーネントを追加して使う前提でまとめています。

Reference

詳細コンポーネントリファレンス

検索、左ナビ、用途別カテゴリ、意味、使い方、実装レベル付き。AI もライブラリ名ではなく種類で引けます。

開く

Workflow

基本の作業手順

  • GameObject を作成する
  • Inspector の下からコンポーネントを追加する
  • 値を設定して Play で実行確認する

Level

完成度の見方

  • 実装済み: 実際に使える機能
  • 一部実装: 基本動作はあるが Unity 同等ではない機能
  • 表示段階: 追加と Inspector 表示が中心の機能

Runtime

Play で動く主なもの

  • Jolt Physics による 3D 重力、Rigidbody、当たり判定
  • PlayerInput の Move / Jump / Fire 基礎処理
  • BehaviorTree / GOAP / 経路探索 / 操舵 / Python AI の実行

Editor

編集で使う主なもの

  • Hierarchy / Scene / Game / Inspector / Project / Console
  • 移動、回転、拡縮ギズモ
  • Scene 保存、名前を付けて保存、Prefab 保存

Assets

読み込み対象

  • PNG / JPG / WAV / OBJ / FBX
  • AI 用の PY / XML / JSON / ONNX
  • FBX SDK を使ったモデル読み込みの入口
  • メッシュ当たり判定、NavMesh 障害物化の入口
