How To Read

このリファレンスの見方

CG2 は GameObject にコンポーネントを追加して機能を増やすエディタです。 各項目は「意味」「使い方」「実装レベル」を先に書き、内部ライブラリ名は必要な場合だけ補足します。 そのため、AI も「BehaviorTree.CPP」ではなく「行動制御」「状態管理」「経路探索」のように用途から探します。

基本ワークフロー

#
  1. Hierarchy または GameObject メニューから GameObject を作ります。
  2. Inspector の一番下にある「コンポーネントを追加」から必要な機能を足します。
  3. Transform、描画、当たり判定、Rigidbody、入力、AI などの値を設定します。
  4. Play を押すと、物理、入力、スクリプト、AI の実行系が動きます。
  5. Stop を押すと、Play 前の Scene 状態へ戻します。
重要: 描画だけのコンポーネントは見た目を出すものです。物理で動かしたい場合は Rigidbody と当たり判定も必要です。

実装レベル

#
表記意味

Play の考え方

#

Play は編集状態の Scene を実行状態へ切り替えます。Play 中は Jolt Physics の固定更新、入力、AI、Navigation、Script の入口が動きます。 Stop 時は Play 前の Scene に戻すため、実行中に落下した Rigidbody や移動した AI の位置は編集状態へ復元されます。 

Play を押す
  ↓
Scene のバックアップを作る
  ↓
FixedUpdate で物理を進める
  ↓
Update で入力 / AI / Script を進める
  ↓
Draw で Scene View / Game View を描画する
  ↓
Stop でバックアップへ戻す

GameObject / Transform

#

GameObject

実装済み

意味: Scene 上に置く基本単位です。名前、有効状態、タグ、レイヤー、コンポーネント一覧を持ちます。

使い方: Hierarchy で選択し、Inspector から必要なコンポーネントを追加します。

Transform

実装済み

意味: 位置、回転、拡縮を表します。すべての 3D / 2D 表示と物理の基準です。

使い方: Inspector の X / Y / Z 数値、または Scene ギズモで編集します。

RectTransform / Canvas

表示段階

意味: UI 用の座標と描画範囲です。

使い方: UI コンポーネントを置く土台として追加します。Unity 同等の UI レイアウト実行は未完成です。

カメラ / ライト

#

Camera

一部実装

意味: Game View の視点です。Scene View の編集カメラとは別です。

使い方: GameObject に追加し、Transform で位置と向きを調整します。追従カメラは追従対象が原点にないか確認します。

Light

一部実装

意味: Scene の明るさと色に関係するライトです。

使い方: 色、強さ、向きを設定します。Unity の複数ライト、影、Probe と同等ではありません。

ReflectionProbe / Volume / LightProbe

表示段階

意味: Unity 風の環境表現の入口です。

使い方: Scene 構造として置けます。描画結果への本格反映は限定的です。

アセット配置

#

Project には `resources` 配下の画像、音声、OBJ、FBX などが表示されます。基本形は GameObject メニューから作るものなので、Project へ余計に並べる必要はありません。

種類意味注意
PNG / JPGスプライトやテクスチャの元データです。任意 Material 割り当てはまだ限定的です。
OBJ静的モデルの読み込みに使います。複雑な Material は限定対応です。
FBXBlender などから出したモデルの読み込みに使います。FBX SDK の読み込み入口があります。スキン、ボーン、Animation Clip はまだ限定的です。
WAVAudioSource の音源候補です。Audio Mixer ほどの管理は未対応です。

Scene / Prefab 保存

#
  • 保存: 現在開いている Scene に上書き保存します。
  • 名前を付けて保存: 保存先と Scene 名を選んで保存します。
  • 読み込み: ファイル選択で Scene を読み込みます。
  • Prefab: 選択 GameObject を再利用できるデータとして保存します。

Unity の Prefab Apply / Revert、ネスト Prefab、差分表示までは未対応です。

3D 物理

#

3D 物理は Jolt Physics 5.5.0 を使います。GameObject に Rigidbody と当たり判定を付けると、Play 中に重力、速度、衝突、反発、摩擦の基礎処理が動きます。

コンポーネント意味使い方レベル
Rigidbody質量、速度、重力、減衰、固定軸を持つ物理本体です。落下、押し返し、力、トルクで動かしたい物体に付けます。一部実装
Box Collider箱の当たり判定です。床、壁、箱に使います。Center / Size を見た目に合わせます。実装済み
Sphere Collider球の当たり判定です。球、ボール、丸い敵に使います。Radius を見た目に合わせます。実装済み
Capsule Colliderカプセルの当たり判定です。キャラクター、柱、丸みのある物体に使います。一部実装
Mesh Colliderモデルの三角形に近い形で当たり判定を取る入口です。FBX / OBJ の穴や複雑形状を使う場合に選びます。重いモデルは頂点削減が必要です。一部実装
Physics Material摩擦、静止摩擦、反発を決める設定です。氷、ゴム、金属床、跳ねる球などの差を作る時に使います。一部実装
Character ControllerRigidbody 任せではないキャラクター用移動です。プレイヤーや敵を安定して歩かせたい場合に使います。一部実装
Joint物体同士を制約でつなぎます。扉、ばね、固定接続、振り子の入口です。一部実装
使い分け: 動く物体には Rigidbody + Collider。動かない床や壁は Collider だけ、または Static 扱いで置きます。

メッシュ当たり判定

#

円やドーナツのように、箱では形が合わないモデルは Mesh Collider を使います。 モデルの頂点と三角形を使って当たり判定を作るため、穴のある形状は穴として扱えます。

  • 静的メッシュ: 床、壁、ステージ、障害物に向いています。
  • 動的メッシュ: 動かすと重くなりやすく、貫通や震えが出やすいため頂点削減が必要です。
  • NavMesh 障害物: MeshFilter / MeshCollider の見た目サイズを使って経路探索の障害物にします。

入力

#

Input

一部実装

意味: 簡易的な移動入力設定です。

使い方: 移動速度、前後左右、ジャンプキーなどを設定して Play します。

PlayerInput

一部実装

意味: ActionMap / Action / Binding を使う Unity Input System 風の入力です。

使い方: Create Actions... で .inputactions を作り、Actions に割り当て、Default Map と Event 名を設定します。

EventSystem

表示段階

意味: UI のクリック、選択、ホバーなどの入力イベントの入口です。

使い方: UI を実行する Scene に置きます。本格 UI イベント実行は未完成です。

例: W で前進させる最低構成

1. Player の GameObject を作る
2. Rigidbody を追加する
3. Box Collider または Capsule Collider を追加する
4. PlayerInput を追加する
5. Create Actions... で InputActionAsset を作る
6. Player / Move に W/S/A/D を割り当てる
7. Play を押す

Move はキーそのものではなく「移動という行動」として扱う。

C++ スクリプト

#

C++ スクリプトは GameObject に処理を追加するための入口です。 Unity の MonoBehaviour と同じ考え方で、Start / Update / FixedUpdate / OnDestroy / OnCollision / OnTrigger のような呼び出し口を作る対象です。

概要 作成手順 API 一覧 構造体の使い方 入力 物理 AI センサー コード例
概要と DLL の役割
項目意味現状
StartPlay 開始時に 1 回だけ呼びます。一部実装
Update毎フレームの通常更新です。一部実装
FixedUpdate物理用の固定更新です。一部実装
OnCollision / OnTrigger当たり判定イベントです。一部実装
DLL リロード外部 C++ をビルドして差し替える入口です。一部実装
生成場所: C++ スクリプトを新規作成すると、resources/scripts/スクリプト名/ に .h / .cpp / build_debug.bat / build_release.bat が UTF-8 BOM 付きで生成されます。 DLL は通常 x64/Debug/スクリプト名.dll に出ます。
作成手順
  1. Project で C++ Script を作り、名前を決めます。
  2. 生成された .cpp を開き、EditorScript_Start と EditorScript_Update などを書きます。
  3. build_debug.bat を実行して DLL をビルドします。
  4. GameObject を選び、Inspector の一番下から Script または MonoBehaviour を追加します。
  5. そのコンポーネントのアセット欄へ、生成された .dll を割り当てます。
  6. Play を押すと、DLL が読み込まれ、Start / Update / FixedUpdate / PhysicsEvent が実行されます。

hasDetailsPython / 画像 / 音声 / 動きlabel、text、motion などの詳細値が有効なら true です。
confidence検出結果の強さを見たい時検出の信頼度です。0.0f から 1.0f で扱います。
視界 / 接続対象で見る値使うセンサー意味
connectedGameObjectId視界、動き検出、接続対象があるセンサーInspector で接続した対象 GameObject ID です。
detectedGameObjectId視界、接続対象ベースの検出実際に検出した GameObject ID です。外部 Python / 画像 / 音声結果だけの場合は -1 のままです。
range視界、距離判定、範囲判定距離、半径系の設定値です。
angleDegrees視界視野角です。
distance視界、接続対象があるセンサー対象までの距離です。
direction視界、対象へ向く処理対象へ向かう方向です。x / y / z を読みます。
画像 / 物体検出で見る値使うセンサー意味
label画像物体検出、色追跡、Python 検出物体名、顔、色名などの検出ラベルです。例: person、red。
screenPosition画像物体検出、色追跡画面上の中心座標です。Python 出力の x / y から入ります。
boundsPosition画像物体検出検出枠の左上座標です。Python 出力の boundsX / boundsY から入ります。
boundsSize画像物体検出検出枠の幅と高さです。Python 出力の w / h から入ります。
動き検出で見る値使うセンサー意味
motion動き検出動きの方向です。x は横方向、y は縦方向またはワールド Z 差分です。
motionMagnitude動き検出動きの大きさです。この値がしきい値以上なら「動いた」と判断します。
音声で見る値使うセンサー意味
textWhisper 音声認識音声認識文字列です。例: 進め、止まれ。
command音声コマンドゲーム用に正規化したコマンド名です。例: Move、Stop、Attack。
commandId音声コマンド文字列比較を避けたい時のコマンド番号です。Python 出力の commandId から入ります。
読み方: isDetected だけ見れば最低限動きます。 ただし、何が付いていないのか切り分ける時は hasComponent、 Inspector で無効になっていないか見る時は isActive、 誰を見ている想定か確認する時は connectedGameObjectId を使います。

1|text=進め|command=Move|commandId=1|confidence=0.95
1|motionX=0.4|motionY=0.0|motionMagnitude=0.4|confidence=0.8

// 1. 関数から構造体を受け取る

float moveY = moveInput.y;

force.z = 0.0f;

extern "C" __declspec(dllexport) void EditorScript_OnPhysicsEvent(
	int32_t gameObjectId,
	const EditorScriptPhysicsEvent* physicsEvent) {
	if (runtimeApi == nullptr || physicsEvent == nullptr) {
		return;
	}

	if (physicsEvent->type == EditorScriptPhysicsEventTypeCollisionEnter) {
		int32_t otherId = physicsEvent->otherGameObjectId;
		float hitX = physicsEvent->point.x;
		float hitY = physicsEvent->point.y;
		float hitZ = physicsEvent->point.z;
		float normalY = physicsEvent->normal.y;

		(void)gameObjectId;
		(void)otherId;
		(void)hitX;
		(void)hitY;
		(void)hitZ;
		(void)normalY;
	}
}

AI コンポーネント全体

#

AI は「ライブラリ名」ではなく「何をする AI か」で選びます。 実行用は Agent / Planner / Sensor です。Blackboard、Goal、Action、Task などは設定や判断材料として使います。

選び方: 追跡や待機など単純な動きは「行動制御」か「状態管理」。目的から行動順を決めたい時は「目標計画」。経路が必要なら「経路探索」。なめらかな移動方向を作りたい時は「移動操舵」です。
AI アセット設定: Project で `.py` / `.xml` / `.json` / `.onnx` を選び、Inspector の `AI アセットパス` または `選択中 AI アセットを設定` を使います。 `.py` は方向 AI なら `x y z`、センサーなら `0` または `1` を標準出力へ返します。
C++ スクリプト連携: AI コンポーネント単体で完結するものと、C++ Script から反応を読む前提のものがあります。 視界、画像物体検出、画像色追跡、動き検出、Whisper 音声認識、音声コマンドは runtimeApi->GetAiSensorState() で現在の反応状態を取得できます。

FeelKit

#

FeelKit は触覚、画面揺れ、演出、音連動などのゲームフィールを扱うための入口です。 CG2 では HapticSource を GameObject に付け、振動強度、時間、ループなどを設定する形に寄せています。

コンポーネント意味使い方レベル
